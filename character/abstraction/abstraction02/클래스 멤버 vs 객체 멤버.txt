클래스와 객체가 모두 같은 값을 가지는 경우는?

예시)
    미키마우스의 꼬리는 몇 개인가요?
    제리의 꼬리는 몇 개인가요?
    쥐의 꼬리는 몇 개인가요?

    answer : 1게;

위와 같이 클래스 단계에서 모두 같은 값을 가질 경우는 클래스에서도 1개 라고 정의 할 수 있다.
만약 해당 값을 힙 영역의 Mouse객체들에게 분담 해준다면 각자의 꼬리 변수를 가지고 있어 쓸데 없이 메모리가 낭비 될 수 있다.

힙 메모리에서는 같은 클래스에서 나와도 다른 메모리 주소를 가진다. (new 연산자를 통해 클래스의 인스턴스를 생성할 때 기준)

이럴 때 에는 클래스에서 static 예약어를 사용하여 모든 인스턴스가 같은 메모리 주소를 바라보게 한다.
이때 static이 붙은 변수는 힙 영역의 메모리 공간을 가지는게 아니라 스태틱 영역(클래스 영역)에서 공간을 가지게 된다.


클래스 멤버 vs 객체 멤버

클래스 멤버들은 static 키워드와 함께 사용되며 메모리 구조에서 static영역(클래스 영역) 에엇 상주하게 되며 static(정적) 멤버라고도 한다.
객체 멤버들은 객체가 클래스의 인스턴스 이므로 인스턴스 멤버라고 한다.

클래스 멤버 = static 멤버 = 정적 멤버
객체 멤버 = 인스턴스 멤버

정적 멤버 속성들의 예시
    * 사람 클래스의 인구
    * 고양이 클래스의 다리 개수
    * 승용차 클래스의 바퀴 개수
    * 보병 클래스의 주특기번호
    * 남자 클래스의 주민등록번호 성별 코드
    * 박카스 클래스의 제조사명, 용량, 성분

이처럼 정적 멤버는 해당 클래스의 모든 객체가 같은 값을 가질 때 사용한다. 물론 이외의 경우도 있지만 그때는 정당한 논리를 가지고 사용해야한다.


정적 멤버를 사용하기에 좋은 방법
정적 멤버들은 객체가 아닌 클래스에 속해 있다.
클래스는 JVM 구동 시  메모리의 스태틱 영역에 바로 배치 되기 때문에 객체의 존재 여부에 관계 없이 쓸 수 있다.
main() 메소드는 항상 정적이어야 한다. 메모리가 초기화 된 순간 객체는 하나도 존재하지 않기 때문에 객체 멤버 메서드를 바로 실행 할 수가 없다.
실무에서는 클래스의 인스턴스를 만들지 않고 사용하게 되는 유틸리티성 메서드를 주로 정적 메서드로 구성한다.

예시 Mat클래스에는 수많은 정적 메서드가 있다.
내가 생각한 예시 DB관련 메소드도 정적으로 구성되어야한다?
 =>  DB는 객체 하나만 존재하면 되고 코드가 실행될 때 마다 객체가 생성되면 db관련 객체가 엄청 많아 질 것이다.
     DB에 연결 할 수 있는 객체를 정적으로 두게 되면 메모리를 아낄수 있지 않을까?

추가적으로 정적 멤버는 스태틱 영역에서 클래스가 배치될 때 메모리 공간을 확보한다.



static 변수 : 스태틱 영역
인스턴스 변수 : 힙 영역
지역 변수 : 스택 영역
